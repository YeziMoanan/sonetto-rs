# Destiny Battle Evidence Audit: Special Attributes And 3088

Date: 2026-07-18
Scope: international 3.6 PC battle only
Baseline commit: `3db3eaa`

This audit records evidence found in the 3.6 runtime data, client Lua, and
the local configuration capture. It does not infer damage formulas, protocol
semantics, or Destiny links that are not present in those sources.

## Client Boundary Notes

- `CharacterEnum.lua` declares `PoisonAddRate=301` but has no corresponding
  `RealHurtRate=217` field. `CharacterDestinyEnum.lua` lists `301` among
  special Destiny attributes but does not list `217`.
- The fight effect enum values for real-hurt/poison effects are a separate
  namespace from character attribute IDs `217` and `301`; matching numbers
  must not be treated as a protocol alias.
- The local runtime capture manifest contains eight UTF-16 configuration
  resources and hashes. It contains no battle packet, power-info, or damage
  trace fixture. `source.json` stores only relative source identity and
  hashes, with no absolute path.

## Required Evidence Before Implementation

1. A redacted 3.6 battle fixture showing a caster/target pair and the damage
   result with a non-zero `217` or `301` value, plus the associated attribute
   state.
2. A redacted effect-295 payload with omitted `num` and/or `max`, and the
   resulting client state, to distinguish replacement, deletion, and default
   behavior.
3. A 3.6 Destiny or client execution trace that links the 3088 bonus IDs to
   a Destiny facet, ex-level row, or explicit trial/activity role.

Until these artifacts exist, the corresponding tests should remain explicit
`not_ready` coverage and no damage/protocol behavior should be guessed.
