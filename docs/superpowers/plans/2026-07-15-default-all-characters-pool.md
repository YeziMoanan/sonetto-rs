# Default All-Characters Pool Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Generate a standalone international 3.6 `summon.json` where official standard pool `2` contains all 120 online characters, with a deterministic PowerShell generator and black-box verifier.

**Architecture:** A strict PowerShell 5.1 generator parses the official `summon` and `character` tuples, derives online IDs by rarity, replaces only the five pool-2 rows, verifies a sibling temporary file, and atomically publishes the output. A black-box test uses the real sources, proves source immutability and exact membership, checks deterministic regeneration, and compares the checked-in artifact byte for byte.

**Tech Stack:** PowerShell 5.1, UTF-8 JSON, Node.js only for an independent final artifact check

**Repository constraint:** Do not commit, push, create a PR, deploy the generated file, restart services, or modify any runtime/database path. Checkpoints use focused tests and `git diff --check`.

## Task 2: Strict Deterministic Generator

**Files:**
- Create: `tools/banner-scheduler/New-DefaultAllCharactersSummon.ps1`

- [ ] **Step 1: Define inputs and strict UTF-8 helpers**

Use mandatory `SummonJson`, `CharacterJson`, and `OutputPath` strings plus `Force`. Resolve both existing inputs and normalize the possibly absent output with `[IO.Path]::GetFullPath`. Reject case-insensitive equality between output and either source before any write.

Read JSON using:

```powershell
$strictUtf8 = [Text.UTF8Encoding]::new($false, $true)
$text = [IO.File]::ReadAllText($path, $strictUtf8)
$value = $text | ConvertFrom-Json
```

Validate tuple names, record arrays, positive unique character IDs, integer rarities `1..5`, and exactly one baseline pool-2 row per rarity.

- [ ] **Step 2: Build replacement rows**

For rarities `5,4,3,2,1`, select character rows with `isOnline -eq "1"`, sort numeric IDs, reject empty groups, and construct ordered objects:

```powershell
[pscustomobject][ordered]@{
    id = 2
    rare = $rarity
    summonId = ($ids -join "#")
    luckyBagId = ""
}
```

Iterate source summon rows. At the first pool-2 row append all replacements; skip every source pool-2 row; append all other records unchanged.

- [ ] **Step 3: Serialize and publish safely**

Serialize records through the PowerShell pipeline so PowerShell 5.1 emits an array, then wrap it explicitly as the `summon` tuple:

```powershell
$recordsJson = $outputRecords | ConvertTo-Json -Depth 10
$outputJson = "[`r`n  `"summon`",`r`n$recordsJson`r`n]`r`n"
```

Write strict UTF-8 without BOM to a unique temporary sibling. Parse the temporary output and independently recheck tuple name, row count, pool-2 rarity order, 120 unique eligible IDs, and exclusion of `3029` and `9998`.

If the destination exists without `-Force`, throw and remove the temporary file. If it exists with `-Force`, publish with `[IO.File]::Replace`; otherwise use `[IO.File]::Move`. Print total and per-rarity counts.

- [ ] **Step 4: Run the black-box test**

Run the Task 1 command. Expected: progress reaches the final artifact check and FAILS because the checked-in artifact does not exist yet; all preceding generator assertions pass.
